# Coding Agent Interaction History (Ringle Frontend)

본 문서는 과제 수행 과정에서 코딩 에이전트(Codex, Copilot)를 **실무 방식으로 활용**한 기록입니다.

- 요구사항 해석 → 설계 → 구현 → 검증까지의 흐름을 투명하게 남겼습니다.
- **툴 입력 방식**, **검토/수정/검증 방법**를 포함합니다.

## 0) 스펙 우선 설계

- 백엔드 API 명세(API.md)를 기준으로 화면 흐름과 페이지 라우팅을 먼저 정의했습니다.
- SSE 포맷(메타/청크/완료)을 기준으로 스트리밍 UI 업데이트 방식을 설계했습니다.

---

## 1) 사용자 입력 (요청 내용 요약)

> 본 섹션은 **사용자가 에이전트에게 실제로 입력한 요청 유형**을 요약합니다.

### 1.1 요구사항 기반 요청

- "홈 카드/헤더/레이아웃 구조 정리"
- "Chat UI 음성 녹음/재생 UX 구성"
- "멤버십 권한에 따른 접근 제어 및 비활성화 UI"
- "SSE 응답을 파싱해 스트리밍 텍스트 렌더링"

### 1.2 검증/실행 요청

- "음성 녹음 버튼 동작 확인"
- "스크롤 자동 내려감/overflow 처리"
- "권한 없는 접근 시 404 리다이렉트"

### 1.3 문서화 요청

- "디렉토리 구조 문서화"

### 1.4 사용자 경험/에러 처리 요청

- "짧은 녹음은 에러 메시지로 안내"
- "비영어 입력 시 안내 문구 표시"
- "스트리밍 텍스트 word 단위 렌더링"

---

## 2) 에이전트 처리 흐름

### 2.1 코드 생성

- 레이아웃/스타일 변경은 `styled-components` 기반으로 국소 변경
- chat feature는 hook/컴포넌트 분리 후 UI 및 상태 로직을 연결
- SSE 파싱 로직을 이벤트 단위로 처리하도록 개선

### 2.2 검토

- 화면별 컴포넌트 분리 정도와 재사용성 확인
- 비동기 처리(녹음/스트리밍/재생)의 상태 충돌 점검
- 오디오 Blob URL 생성/해제 시점 점검

### 2.3 수정

- 메시지 스크롤 자동 유지와 overflow 고정 처리
- 녹음 길이 1초 미만 가드 및 UX 메시지 추가
- "영어로 말해주세요" 등 사용자 친화 메시지 정제
- SSE 응답(JSON) 파싱으로 chunk만 이어붙이도록 처리

### 2.4 검증

- 생성된 코드 내용을 직접 확인하고 검증
- 개발 서버에서 직접 UI/UX 시나리오 확인
- 네트워크 탭에서 SSE 청크 수신/파싱 흐름 확인
- 콘솔 에러 및 권한 리다이렉트 동작 점검

---

## 3) 도구 활용 및 책임 분담

- Codex: 코드 구조 정리, 스트리밍 파싱 로직 개선, UI 상태 설계
- Copilot: 타입/컴포넌트 리팩토링 보조, 반복 코드 정리
- 사용자: 실제 화면 확인 및 UX 판단, 문구/흐름 피드백
